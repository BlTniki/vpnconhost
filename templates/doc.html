Документация vpnconservice
Перед запуском приложения в папке с проектом должен лежать файл appconf.txt в котором построчно должы быть указаны следующие поля:
Пароль указываемый в заголовке запроса "Auth" : (любая строка) Рабочую дирректорию в которой лежит проект. Пример: /ur/work/dir/ (Обязательно / в конце) Адресс сервера, на котором запускается проет, и порт, который слушает wireguard. Пример: 0.0.0.0:0000 Ответ на вопрос: "Запуск в тестовом режиме?(YES/NO)"
Аутентификация
В каждом запросе должен присутствовать ключ "Auth" со значением в виде пароля, который был указан в appconf.
Доступные пути /api/1.0/:
  • /api/1.0/peers: (Методы: GET, POST)
  • /api/1.0/peers/{peerId}: (Методы: GET, PUT, DELETE)
  • /api/1.0/conf:{peerId} (Методы: GET, POST)
  • /api/1.0/logs: (Методы: GET, DELETE)
  • /api/1.0/appconf: (Методы: GET)
  • /api/1.0/doc: (Методы: GET)
    • /api/1.0/peers:
    Метод GET:
    Возвращает список всех пользователей в виде json с кодом 200.
    Метод POST:
    Создание нового пользователя. В запросе должно присутствовать тело:
    peerId должен быть строкой,a также должен содержать исключительно латиницу и цифры.
    peerIp должен быть целым числом от 1 до 254 включительно.
    Ответ:
    При корректном запросе возвращает пира в виде json код 200
    При некорректном запросе возвращает ответ в виде json код 4ХХ
